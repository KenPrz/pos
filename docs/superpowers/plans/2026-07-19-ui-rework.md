> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Both frontends rebuilt on Tailwind v4 + shadcn/ui in the IBM/Carbon language — the back office in the Plate layout (with the new Today landing), the register in a purpose-built fast/legible till language — with every EXISTING label, flow, and behavior byte-identical.

Shared components between the two apps are byte-identical copies (like `money.ts` — two files, one shape, review-verified). Spec: `docs/superpowers/specs/2026-07-19-ui-rework-design.md`. Design authority: root `DESIGN.md` (IBM/Carbon).

**Tech Stack:** Tailwind v4 (CSS-first `@theme`), shadcn/ui, `@fontsource/ibm-plex-sans`, Next.js 16, React 19, TS 7 (tsgo), vitest/@testing-library.

## The component vocabulary (contracts — foundation tasks implement, screen tasks consume)

**`src/components/ui/` (shadcn, Carbon-ized; byte-identical across apps):**
- `Button` — variants `primary` (blue, hover `blue-hover`, active `blue-80`) / `secondary` (ink bg, inverse-ink text) / `tertiary` (canvas bg, 1px primary border, primary text) / `ghost` (primary text, transparent; surface-1 hover) / `danger` (error bg); sizes `default` (12px×16px pad, 14px/400 label) / `lg` (register action rows, min-h 56px) / `xl` (register action zone, min-h 64px, 18px label). Radius 0, no shadow, focus = 2px primary outline.
- `Input` / `Textarea` — surface-1 fill, no side borders, 1px bottom hairline; focus swaps to 2px primary bottom border; `aria-invalid` swaps to 2px error. Radius 0.
- `Select`, `Checkbox`, `Tabs` (selected = 2px primary bottom border + weight 600), `Table` (thin wrapper: hairline row borders, sentence-case `ink-muted` body-sm headers, optional zebra), `Dialog` (square, hairline border, flat scrim rgba(22,22,22,.5), no shadow), `Sheet` (right panel, square, hairline left border), `Card` (canvas bg, 1px hairline, radius 0; `elevated` variant = surface-1 bg), `Badge` (the ONE pill: variants `success`/`warning`/`error`/`info`/`neutral`, caption type).

**App-level composites:**
- Both apps: `StatusPill` (dot + sentence-case label, semantic color), `EmptyState` (icon-less: card-title line + body-sm ink-muted line), `ConfirmDialog` (Dialog composition: message, Cancel ghost, confirm `danger`|`primary`; replaces `window.confirm` call sites — SAME copy strings passed through).
- Back office: `AppSidebar` (brand, `LocationSwitcher`, `NavGroup`/`NavItem` with eyebrow + 2px blue left indicator + optional count badge, footer user slot), `SectionHeader` (display-md 300 title, ink-muted subline, right action slot), `StatCard` (label caption, big weight-300 value, meta line), `DataTable` (Table + toolbar slot + EmptyState wiring), `FieldRow` (label + control + error line, Carbon spacing).
- Register: `ActionZone` (fixed bottom full-width bar; ONE `Button xl` primary or danger + optional ghost secondary), `MoneyText` (tabular-nums, sizes `line` 18px / `total` 42px w300), `CartLine` (name, qty, modifiers indent, line total; void affordance slot), `TileButton` (≥96px hairline tile for menu/floor: title, meta lines, optional left status edge in semantic color), `PillStrip` (split-check strip: `Badge`-based chips, active/settled states), `PrepChip` (48px status chip cycling held/fired/ready colors).
