# Docs-to-Wiki Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A five-chapter User Manual in `docs/manual/` plus a CI job that mirrors the manual and the numbered technical docs into the GitHub Wiki (sidebar, landing page, link rewriting) on every `main` push.

**Architecture:** Prose first, machinery second. Chapters are authored from shipped behavior with every UI label and command verifiable against the repo; the sync is one committed bash script (`scripts/wiki-sync.sh`) that builds the wiki tree deterministically and is runnable locally, plus a thin workflow that pushes it. Spec: `docs/superpowers/specs/2026-07-19-docs-wiki-design.md`.

**Tech Stack:** Markdown, bash + sed/awk, GitHub Actions. No new toolchain.

## Global Constraints

- Branch `docs-wiki` (checked out). Commits `Docs: <what>`, NO co-author trailer.
- **Manual voice contract (binding for every chapter):** imperative, second person ("Tap **PAY CASH**"), one task per `##` section, "what you see" before "what to do", numbered steps for procedures, a `> Note:` blockquote for sharp edges. **Every quoted UI label, make target, env var, and error message must literally exist in the repo** — verify by grep before writing it, cite nothing you can't point to. No screenshots. No invented features. American English, sentence-case headings.
- Chapter files carry an H1 matching the spec's wiki page names (`# Getting Started`, `# Cashier Guide`, `# Supervisor Guide`, `# Manager Guide`, `# Operator Guide`). Cross-references between manual chapters and to numbered docs use plain relative links (`02-supervisor-guide.md`, `../03-api.md`) — the sync rewrites them.
- `docs/superpowers/**` never appears in the wiki output.
- Prose verification standard (replaces TDD for authoring tasks): a **label audit** step — extract every backticked/bolded UI string and command from the chapter and grep it against the source tree; zero misses before commit.
- Machine-local: dev stack via `make` targets; full suites NOT required (no app code changes anywhere in this plan).

### Task 4: The sync — `scripts/wiki-sync.sh` + `.github/workflows/wiki.yml` + pointers

**Files:**
- Create: `scripts/wiki-sync.sh`, `.github/workflows/wiki.yml`
- Modify: `docs/README.md` (manual listed in the index), `CLAUDE.md` (one line under docs pointing at the wiki + "edit in repo, wiki is generated")
- Test: local dry run + link audit (steps below)

**Interfaces:** Consumes the seven numbered docs + five manual chapters. Produces the wiki tree.

- [ ] **Step 1: The script**

```bash
#!/usr/bin/env bash
# scripts/wiki-sync.sh OUT_DIR — build the GitHub-wiki tree from docs/.
# CI pushes the result to <repo>.wiki.git; run locally for a dry preview.
# Synced pages are OVERWRITTEN on every run — the wiki is generated, edit docs/ instead.
set -euo pipefail

OUT="${1:?usage: wiki-sync.sh OUT_DIR}"
mkdir -p "$OUT"

# source-path → wiki page name (flat namespace; names are the H1s, hyphenated)
declare -A PAGES=(
  [docs/manual/00-getting-started.md]="Getting-Started"
  [docs/manual/01-cashier-guide.md]="Cashier-Guide"
  [docs/manual/02-supervisor-guide.md]="Supervisor-Guide"
  [docs/manual/03-manager-guide.md]="Manager-Guide"
  [docs/manual/04-operator-guide.md]="Operator-Guide"
  [docs/00-overview.md]="Overview"
  [docs/01-architecture.md]="Architecture"
  [docs/02-data-model.md]="Data-Model"
  [docs/03-api.md]="API"
  [docs/04-backend-conventions.md]="Backend-Conventions"
  [docs/05-rbac.md]="RBAC"
  [docs/06-roadmap.md]="Roadmap"
)

REPO_URL="${WIKI_REPO_URL:-https://github.com/KenPrz/pos}"
SHA="$(git rev-parse --short HEAD)"

# Build one sed program that rewrites every known relative link to its wiki page,
# in ]( ... ) position, tolerating ./ ../ docs/ manual/ prefixes and #anchors.
SED_ARGS=()
for src in "${!PAGES[@]}"; do
  base="$(basename "$src")"
  page="${PAGES[$src]}"
  SED_ARGS+=(-e "s|](\\(\\./\\)\\?\\(\\.\\./\\)*\\(docs/\\)\\?\\(manual/\\)\\?${base}|](${page}|g")
done

for src in "${!PAGES[@]}"; do
  [ -f "$src" ] || { echo "missing source: $src" >&2; exit 1; }
  sed "${SED_ARGS[@]}" "$src" > "$OUT/${PAGES[$src]}.md"
done

# Home: docs/README.md with the same rewrites, plus a manual pointer up top.
{
  echo "> **New here?** Start with [Getting Started](Getting-Started) — the User Manual for people running the store. This page indexes the technical documentation."
  echo
  sed "${SED_ARGS[@]}" docs/README.md
} > "$OUT/Home.md"

cat > "$OUT/_Sidebar.md" <<'EOF'
**User Manual**
- [Getting Started](Getting-Started)
- [Cashier Guide](Cashier-Guide)
- [Supervisor Guide](Supervisor-Guide)
- [Manager Guide](Manager-Guide)
- [Operator Guide](Operator-Guide)

**Technical Documentation**
- [Overview](Overview)
- [Architecture](Architecture)
- [Data Model](Data-Model)
- [API](API)
- [Backend Conventions](Backend-Conventions)
- [RBAC](RBAC)
- [Roadmap](Roadmap)
EOF

printf '*Synced from [`docs/`](%s/tree/main/docs) at %s — edit in the repo, not here.*\n' "$REPO_URL" "$SHA" > "$OUT/_Footer.md"

# Any remaining relative .md link is a link to an unsynced file — rewrite to the repo blob.
sed -i "s|](\\(\\./\\)\\?\\(\\.\\./\\)\\+\\([^)]*\\.md\\)|](${REPO_URL}/blob/main/\\3|g" "$OUT"/*.md

echo "wiki tree built in $OUT ($(ls "$OUT" | wc -l) files)"
```

> The sed patterns above are the intent; validate each against real link instances
> (grep `](` across the sources) and adjust escaping until the dry-run audit (Step 3)
> is clean — the audit is the authority, not the sketch. If a rewrite case appears
> that sed handles miserably, a small awk block is acceptable; no new toolchain.

- [ ] **Step 2: The workflow**

```yaml
# .github/workflows/wiki.yml
name: Wiki

on:
  push:
    branches: [main]
    paths: [docs/**, scripts/wiki-sync.sh]
  workflow_dispatch:

permissions:
  contents: write

jobs:
  sync:
    name: Sync docs to wiki
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v5

      - name: Build wiki tree
        run: bash scripts/wiki-sync.sh /tmp/wiki-out

      - name: Push to wiki
        run: |
          git clone "https://x-access-token:${GITHUB_TOKEN}@github.com/${GITHUB_REPOSITORY}.wiki.git" /tmp/wiki
          rsync -a --delete --exclude .git /tmp/wiki-out/ /tmp/wiki/
          cd /tmp/wiki
          git config user.name "wiki-sync"
          git config user.email "wiki-sync@users.noreply.github.com"
          git add -A
          git diff --cached --quiet && { echo "wiki unchanged"; exit 0; }
          git commit -m "Sync from ${GITHUB_SHA::7}"
          git push
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```
